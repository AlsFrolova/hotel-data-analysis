**Анализ отмен бронирований гостиничного бизнеса** – исследование факторов, влияющих на отмены (тип отеля, тип клиента, lead time, длительность, география), сегментация клиентов и разработка рекомендаций для снижения убытков.

---

## 🎯 Задача

Провести комплексный анализ данных об отельных бронированиях:

- Оценить общий **процент отмен** и его зависимость от типа отеля, типа клиента, сезона.
- Выявить различия в бронировании **City Hotel** и **Resort Hotel**.
- Проанализировать влияние **lead time** (время между бронированием и прибытием) на частоту отмен.
- Оценить влияние **длительности пребывания**, **страны гостя**, **типа номера**.
- Найти **аномалии** и объяснить их причины.
- Сформулировать бизнес-рекомендации для снижения отмен и увеличения выручки.

---

## 📊 Данные

### Таблица `bookings`

| Колонка                      | Описание |
|------------------------------|----------|
| `hotel`                      | Тип отеля (City Hotel / Resort Hotel) |
| `is_canceled`                | Бронирование отменено (1) или нет (0) |
| `lead_time`                  | Дней между бронированием и датой прибытия |
| `arrival_full_date`          | Полная дата прибытия |
| `arrival_date_year`          | Год прибытия |
| `arrival_date_month`         | Месяц прибытия |
| `stays_total_nights`         | Общее число забронированных ночей |
| `adults` / `children` / `babies` | Количество гостей |
| `meal`                       | Тип питания |
| `country`                    | Страна происхождения клиента |
| `reserved_room_type`         | Зарезервированный тип номера |
| `customer_type`              | Тип бронирования (Transient, Contract, Group, Transient-Party) |
| `reservation_status`         | Последний статус (Check-Out, Canceled, No-Show) |

> **Общий объём:** 119 390 записей, охватывающих период с июля 2015 по август 2017.

## 🔬 Этапы анализа

1. **Предобработка**
   - Приведение столбцов к единому формату (нижний регистр, `_` вместо пробелов).
   - Преобразование форматов.
   - Обработка пропусков.

2. **Исследовательский анализ**
   - Расчёт общего процента отмен и сравнение по типам отелей.
   - Распределение отмен по типам номеров, типам клиентов, странам.

3. **Анализ временной динамики**
   - Построение дневного процента отмен для City Hotel и Resort Hotel.
   - Сглаживание скользящим средним (14 дней) для выявления трендов.

4. **Поиск аномалий**
   - Выявление дня с пиком отмен (август 2016, 235 отмен).
   - Анализ `lead_time` в этот день vs обычные дни.

5. **Факторный анализ**
   - Влияние `lead_time`, `stays_total_nights`, `customer_type`, `country`, типа номера.
   - Визуализация boxplot, violin plot, гистограмм.

6. **Формулировка рекомендаций**
   - Формулирование рекомендаций по снижению отмен.

## 🛠️ Рекомендации (приоритетные меры)

| Приоритет | Мера | Целевой сегмент | Ожидаемый эффект |
|-----------|------|----------------|------------------|
| 🔴 Высокий | Невозвратный депозит 30–50% при бронировании за **>90 дней** | Contract, Transient-Party, City Hotel, lead time >120 дней | Снижение отмен на 15–20% |
| 🔴 Высокий | Бесплатная отмена только за **14–21 день** в летние месяцы | City Hotel, июнь–август | Снижение пиковых отмен |
| 🟠 Средний | Частичная предоплата 20–30% при длительности >7 ночей | Resort Hotel | Снижение риска у семей/групп |
| 🟠 Средний | Дифференцированные тарифы: early‑bird (дешевле, без отмен) | Transient | Улучшение баланса риска |
| 🟡 Низкий | Полная предоплата за 14 дней | Group (групповые) | Снижение массовых отмен |
| 🔍 Аналитика | Пересмотреть договоры с турагентствами | Страны PRT, GBR, FRA | Выявление системных причин |
